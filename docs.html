Overview

The Pinnacle API is a server-side proxy that aggregates odds from The Odds API and surfaces them in a normalized, sanitized format. It runs as Vercel serverless functions at your-domain.vercel.app/api.

All endpoints enforce:

  • Rate limiting — 60 req/min per IP on the odds endpoint; stricter limits on auth endpoints.
  • Strict input validation — every parameter is allowlisted or regex-validated before use.
  • Generic errors — stack traces and upstream URLs are never exposed to clients.
  • Cache-Control: no-store on all API responses.

Base URL

https://your-domain.vercel.app

Response format

All responses are JSON. Errors follow this shape:

{ "error": "invalid_sport" // machine-readable error code }

Self-hosting / local dev

Clone the repo and run vercel dev. Set ODDS_API_KEY in .env.local. The terminal will automatically call /api/odds and display live lines.

Authentication

Pinnacle uses HMAC-SHA256 signed session tokens stored in an __Host-sid HttpOnly cookie. No tokens are ever stored in localStorage or JavaScript-accessible memory.

Sign up flow

# 1. POST credentials curl -X POST https://your-domain/api/auth/signup \ -H "Content-Type: application/json" \ -d '{"email":"you@example.com","password":"your12charpass1"}' # 2. On success: 201 + Set-Cookie: __Host-sid=... { "ok": true }

Cookie attributes

  • HttpOnly — not accessible to JavaScript; prevents XSS token theft.
  • Secure — only sent over HTTPS.
  • SameSite=Strict — prevents CSRF; cookie not sent on cross-site requests.
  • __Host- prefix — requires Secure + no Domain attribute + Path=/.
  • Max-Age=604800 — 7-day expiry.

Password requirements

  • Minimum 12 characters.
  • Must contain at least one letter and one number.
  • Maximum 256 characters.
  • Stored as bcrypt hash with cost factor 12 (~250ms per hash).

API Endpoints

GET /api/odds
Returns sanitized odds for all games in a sport, across all tracked sportsbooks. No auth required.
Parameter
Type
Description
sport
string
Required. One of: basketball_nba, americanfootball_nfl, baseball_mlb, icehockey_nhl, mma_mixed_martial_arts.
region
string
Optional. us (default) or us2.
market
string
Optional. Comma-separated list: h2h, spreads, totals. Default: all three.
Example response
{ "sport": "basketball_nba", "count": 4, "games": [{ "id": "abc123", "home_team": "Boston Celtics", "away_team": "LA Lakers", "commence_time": "2026-05-06T23:30:00Z", "bookmakers": [...] }] }
POST /api/auth/signup
Create a new account. Rate limited to 3 attempts per IP per 10 minutes. Returns a session cookie on success.
Body: { "email": "string", "password": "string" } Success: 201 { "ok": true } + Set-Cookie Error: 400 { "error": "weak_password" } 409 { "error": "email_taken" } 429 { "error": "too_many_attempts" }
POST /api/auth/login
Authenticate an existing account. Rate limited to 5 attempts per IP per 5 minutes. Returns a session cookie on success.
Body: { "email": "string", "password": "string" } Success: 200 { "ok": true } + Set-Cookie Error: 401 { "error": "invalid_credentials" }
POST /api/auth/logout
Invalidates the current session by clearing the session cookie. Always returns 200 regardless of whether the user was logged in.
GET /api/me
Returns the current user’s profile. Requires a valid session cookie. Returns 401 if unauthenticated.
Success: 200 { "id": "uuid", "email": "string", "role": "user", "tier": "pro" }
